Creating a Flask Web Application

This guide will walk through creating a simple application using Flask

The application will exist in a package (Python defines a sub-directory with "__init.py__" file as a package. When importing a package in Python, the "__init__.py" executes.

First, follow the installation page to make a project folder and install Flask within a virtual environment that you have created and activated.

Make sure you are in your project directory with the virutal environemnt active, and create a new package named whatever you like, we'll call it "myapp":

mkdir myapp

Within the package we just created, the __init__.py should look like:

from flask import Flask
myapp=Flask(__name__)
from myapp import routes

We import the Flask class from the flask package so we can instantiate the WSGI application, allowing the sever module to serve Flask applications. The __name__ variable is passed to the Flask class so it can define the starting point from where to load resoueces. Python's __name__ variable is predefined as the module name from which it's used.

We import the routes.py module that we still have to create; it will provide literal routes and view functions to handle them. This import is done at the bottom to avoid circular imports.

The routes.py module should exist in the same package as __init__.py and contain the following:

from myapp import myapp
from flask import render_template
@myapp.route('/')
@myapp.route('/index')
def index():
 return render_template("index.html")

The myapp Flask application instance is imported from the package and the "render_template" function using the Jinja2 template engine, will be imported from flask to convert templates into HTML pages. The @myapp.route lines are function decorators- a Python feature that modifies functions that follow. In this case, the @myapp.route decorator associates its argument (URLs) with the corresponding function. So when the URLs are requested from our application, the functions will be called by Flask and a value returned to the browser sending the request. Both routes ("/" and "/index") are associated with the single view function in our module.
At the top level of our project above the package we've created, Python is required to define the instance of the Flask application (WSGI object). In a script named project.py, we can use a single line to import the instance of the Flask application:

from myapp import myapp

The application is complete, but we need an "index.html" file to render as called in our routes.py module. In the myapp app package create a folder named "templates" and create an HTML file named "index.html".

cd myapp
mkdir templates
nano index.html

The flask framework defaults to searching for HTML template files in the templates folder. The purpose of templates is to seperate the application logic from the web page presentation. In Flask the templates folder stores individual template files and should exist within the application package (e.g., myapp/templates/index.html).

We now have the boilerplate structure to run our very simple web application. To run, Flask must be told how to import the application; this is done by setting the FLASK_APP environment variable to the script defining the application instance. The flask command can then be used to run the application

export FLASK_APP=project.py
flask run

The output should indicate that the server is running on a given IP address, which will be your own computer's localhost and usually on the available port 5000. You can navigate to the webpage from a browser with the URL: "http://localhost:5000/". Hitting Ctrl-C from the terminal you're using will close the connection.
